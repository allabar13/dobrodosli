# Добродошли! 🐦

Интерактивный тренажёр сербского языка (с черногорским режимом) для русскоязычных на Балканах. Уроки из реальной жизни, интервальные повторения, блок «Ошибки» (как в Quizlet), карточки, ачивки и галеб Жарко. Ключевая метрика — выученные слова: никаких стриков и давления. Дизайн «Јадран», светлая и тёмная темы, ставится на телефон как приложение (PWA).

**Вход — только через Google.** Паролей в проекте нет вообще; в данные прогресса персональная информация не записывается (подробности — в конце, раздел «Приватность»).

---

## Запустить локально

Дважды кликни **index.html** — всё работает в браузере, прогресс хранится на устройстве. Google-вход и синхронизация включаются только на опубликованном сайте.

## Опубликовать (один раз настроить — дальше всё автоматом)

### Шаг А · Сайт на GitHub Pages — 5 минут

1. Зарегистрируйся (или войди) на [github.com](https://github.com).
2. Открой Терминал и выполни две команды:
   ```bash
   ~/.local/bin/gh auth login
   ```
   (выбирай: GitHub.com → HTTPS → Login with a web browser — откроется браузер, подтверди код)
   ```bash
   ~/Claude/Приложение/Dobrodosli/publish.sh
   ```
3. Скрипт сам создаст репозиторий, зальёт код и включит хостинг. В конце покажет адрес вида `https://ТВОЙ_НИК.github.io/dobrodosli/` — это и есть продукт. Запиши его, он понадобится ниже.

Дальше **любая правка публикуется этим же скриптом** — `publish.sh` — одна команда, и через минуту сайт обновлён. (Альтернатива без терминала: netlify.com/drop — перетащить папку мышкой, но обновления тогда тоже руками.)

### Шаг Б · Облако Supabase — база для прогресса, 5 минут

1. Зарегистрируйся на [supabase.com](https://supabase.com) (можно через GitHub) → **New project** (имя любое, пароль базы сохрани).
2. Вкладка **SQL Editor** → вставь целиком и нажми **Run**:

```sql
create table if not exists public.progress (
  user_id uuid primary key references auth.users(id) on delete cascade,
  data jsonb not null,
  updated_at timestamptz not null default now()
);
alter table public.progress enable row level security;
drop policy if exists "own row" on public.progress;
create policy "own row" on public.progress
  for all using (auth.uid() = user_id) with check (auth.uid() = user_id);
```

(Этот запрос можно запускать сколько угодно раз — он не ругается на «already exists».)

3. **Project Settings → API**: скопируй **Project URL** и **anon (publishable) key** — вставь их в файл `js/config.js` (поле `window.DOBRODOSLI_SUPABASE`) и снова запусти `publish.sh`. Эти два значения публичные, их можно показывать кому угодно — доступ к данным ограничивает политика из шага 2.

### Шаг В · Вход через Google — 10 минут

1. Открой [console.cloud.google.com](https://console.cloud.google.com) под своим Google-аккаунтом → создай проект (имя любое, например `dobrodosli`).
2. **APIs & Services → OAuth consent screen**: тип **External**, имя приложения «Добродошли», своя почта в контактах → Save. (Публиковать в production можно сразу — скоупы только базовые email/profile.)
3. **APIs & Services → Credentials → Create Credentials → OAuth client ID**:
   - Application type: **Web application**;
   - **Authorized redirect URIs** → добавь: `https://ТВОЙ_ПРОЕКТ.supabase.co/auth/v1/callback` (точный адрес показан в Supabase на странице провайдера Google);
   - Создастся **Client ID** и **Client Secret** — скопируй оба.
4. В Supabase: **Authentication → Sign In / Providers → Google** → включи и вставь Client ID + Secret → Save.
5. Там же: **Authentication → URL Configuration**:
   - **Site URL**: адрес сайта из шага А (например `https://ник.github.io/dobrodosli/`);
   - **Redirect URLs**: добавь тот же адрес и `https://ник.github.io/dobrodosli/index.html`.

Готово: на сайте появится кнопка «Продолжить с Google».

## Как вносить правки (живой продукт)

Контент — человекочитаемый файл `js/data.js`: слова, предложения, ситуации, шутки. Поменяла → `publish.sh` → через минуту у всех пользователей новая версия (сервис-воркер всегда тянет свежее из сети). Просить меня внести правку — тоже валидный путь: я меняю файлы, ты запускаешь публикацию.

## Файлы

- `index.html` — каркас + PWA-подключение
- `css/styles.css` — дизайн «Јадран»: светлая/тёмная тема, 🇷🇸 коралл / 🇲🇪 синий
- `js/data.js` — **контент курса**
- `js/app.js` — экраны и логика; `js/exercises.js` — упражнения
- `js/state.js` — прогресс и повторения; `js/cloud.js` + `js/config.js` — облако
- `js/vendor/supabase.min.js` — локальная копия библиотеки облака
- `manifest.json`, `sw.js`, `icons/` — установка на телефон и офлайн
- `publish.sh` — публикация одной командой
- `design.html` — архив дизайн-стенда (5 китов)

## Приватность — что храним и что нет

- **Не храним:** пароли (их нет в принципе), имена из Google, аватарки, что-либо платёжное.
- **Google-вход:** Google подтверждает личность; почта пользователя остаётся во встроенной системе Supabase Auth (это и есть «аккаунт»), наружу не отдаётся.
- **Таблица прогресса:** только учебные данные — слова/уровни/XP/настройки + имя, которое пользователь сам ввёл для приветствия. Перед отправкой запись чистится от почты (`sanitize` в `js/cloud.js`). Каждый видит только свою строку (RLS-политика).
- **Локально (в браузере):** прогресс и выбранное имя; почта не сохраняется даже локально.
- **Третьи стороны:** Google Fonts (шрифт). Библиотека облака лежит локально в `js/vendor/`.
- ⚠️ Никогда и никуда не вставляй **service_role key** из Supabase — он секретный и всемогущий. В приложении используется только публичный anon key.
